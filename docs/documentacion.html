2. Arquitectura del Sistema

SIGESCON sigue una arquitectura de aplicación web de una sola página (SPA) con un enfoque en el frontend para la interacción del usuario y el manejo de datos locales, utilizando IndexedDB a través de Dexie.js.

Componentes Principales:

  • **Interfaz de Usuario (UI):** Construida con HTML, CSS (Bootstrap 5 y estilos personalizados en documentacion.css) y JavaScript. Se organiza mediante pestañas para navegar entre las diferentes funcionalidades (Resumen General, Gestión de Contratos, Gestión de HES, etc.).
  • **Lógica de la Aplicación:** Implementada en js/script.js. Contiene las funciones para interactuar con la base de datos, manejar eventos del usuario, realizar cálculos (avances físico y financiero), generar reportes (Excel y PDF) y gestionar la interfaz dinámica.
  • **Base de Datos Local:** Se utiliza IndexedDB a través de la librería Dexie.js para almacenar todos los datos de contratos, partidas y HES directamente en el navegador del usuario. Esto permite que la aplicación funcione sin necesidad de un servidor backend.
  • **Librerías Externas:** Se utilizan librerías como Bootstrap 5 (para estilos y componentes UI como el carrusel), Font Awesome (para iconos), Chart.js (para gráficos), SheetJS (xlsx) y jsPDF (para exportación de datos).

Diagrama Conceptual (Texto):

+-------------------+     +---------------------+
| Interfaz de Usuario | --> | Lógica de Aplicación  |
| (HTML, CSS, JS)   |     | (js/script.js)      |
+-------------------+     +---------------------+
         |                         |
         v                         v
+-------------------------+     +--------------------+
| Base de Datos Local     |     | Librerías Externas |
| (IndexedDB via Dexie.js)|     | (Bootstrap, Chart.js, etc.)|
+-------------------------+     +--------------------+

La comunicación principal ocurre entre la Interfaz de Usuario y la Lógica de Aplicación, que a su vez interactúa con la Base de Datos Local y utiliza las Librerías Externas para funcionalidades específicas.

3. Estructura de Archivos y Directorios

El proyecto SIGESCON se organiza en la siguiente estructura de directorios:

/
├── css/
│   ├── ayuda.css
│   └── style.css
├── docs/
│   ├── documentacion.css
│   └── documentacion.html
├── js/
│   └── script.js
├── README.md
└── index.html

Descripción de los archivos y directorios clave:

  • /: Directorio raíz del proyecto.
  • css/: Contiene los archivos CSS para los estilos de la aplicación.
  • css/ayuda.css: Estilos específicos para la página de ayuda.
  • css/style.css: Estilos generales de la aplicación.
  • docs/: Contiene los archivos de documentación.
  • docs/documentacion.css: Estilos para la documentación completa (copia de ayuda.css).
  • docs/documentacion.html: El archivo HTML de la documentación completa.
  • js/: Contiene los archivos JavaScript.
  • js/script.js: El archivo principal con toda la lógica de la aplicación.
  • README.md: Archivo con la descripción general del proyecto.
  • index.html: La página principal de la aplicación.

Guía de Usuario

Introducción a SIGESCON

Figura 1: Vista general del dashboard principal de SIGESCON

SIGESCON es un sistema integral para la gestión de contratos empresariales que te permite:

  • Gestionar contratos y sus partidas
  • Administrar HES (Hojas de Estimación de Servicios)
  • Seguir el avance físico y financiero
  • Generar reportes detallados

Gestión de Contratos

Crear un Nuevo Contrato

Figura 2: Formulario para crear un nuevo contrato

  1. Haz clic en "Nuevo/Editar Contrato" en el menú lateral
  2. Completa los campos requeridos (marcados con *)
  3. Agrega las partidas del contrato
  4. Haz clic en "Guardar Contrato"

Gestionar Partidas

Figura 3: Sección de gestión de partidas del contrato

Para agregar partidas a un contrato:

  1. Haz clic en "Añadir Partida"
  2. Completa la descripción, cantidad y precio unitario
  3. El total se calculará automáticamente

Gestión de HES

Crear una HES

Figura 4: Formulario para crear una nueva HES

  1. Selecciona el contrato asociado
  2. Ingresa el número de HES
  3. Define las fechas de inicio y fin
  4. Selecciona las partidas a ejecutar

Ejecutar Partidas en HES

Figura 5: Sección de ejecución de partidas en HES

Para ejecutar partidas:

  1. Selecciona la cantidad a ejecutar
  2. Verifica los totales calculados
  3. Confirma la ejecución
